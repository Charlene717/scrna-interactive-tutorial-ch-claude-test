高變異基因選擇 (Variable Features)

從數萬個基因中篩選出最具「分辨力」的特徵，聚焦生物學訊號、過濾噪音。

核心概念

為什麼不用所有基因？

典型的 scRNA-seq 數據集包含 2 萬至 3 萬個基因，但大多數基因在細胞間的變異極小——例如管家基因Housekeeping genes：在幾乎所有細胞中穩定表達的基因，如 GAPDH、ACTB在每個細胞都差不多。這些基因對區分細胞類型沒有幫助，反而會增加運算負擔與噪音。

我們需要的是那些在某些細胞中高度表達、在其他細胞中低表達的基因——它們攜帶了區分不同細胞群體的關鍵訊號。這就是高變異基因 (HVGs)Highly Variable Genes：在細胞群體中表達變異顯著高於預期技術噪音的基因。

互動模擬

Mean-Variance 關係圖

下圖模擬了基因的平均表達量與變異度之間的關係。紅色點為被選中的高變異基因。拖動滑桿調整選取數量。

方法比較

選擇方法

方法說明使用場景
VST (Seurat 預設)用局部回歸擬合 mean-variance 關係，取殘差最大的基因NormalizeData 後使用
SCTransform 內建負二項回歸直接輸出殘差方差排名使用 SCTransform 時自動完成
Scanpy 預設按 dispersion（分散度）排名，或 Seurat-flavored VSTScanpy 流程
💡
選多少個？2000–3000 個是常見的經驗值。若你的數據細胞異質性很高（如腫瘤微環境），可適當調高到 3000–5000；若較單純（如純化的某一系 lineage），2000 通常已足夠。
程式碼

實作範例

# 使用 NormalizeData 流程時
pbmc <- FindVariableFeatures(pbmc, selection.method = "vst", nfeatures = 2000)

# 查看前 10 個 HVGs
top10 <- head(VariableFeatures(pbmc), 10)
VariableFeaturePlot(pbmc) # 視覺化

# 注意：若使用 SCTransform，此步驟已自動完成
# 選擇高變異基因
sc.pp.highly_variable_genes(adata, n_top_genes=2000, flavor="seurat_v3")

# 視覺化
sc.pl.highly_variable_genes(adata)

# 後續分析只用 HVGs
adata = adata[:, adata.var.highly_variable]
