STEP 6 / 9

聚類 (Clustering)

在 PCA 空間中建立細胞近鄰網絡，透過社區發現演算法辨識細胞族群。

核心概念

聚類的三步邏輯

建立 KNN 圖

在 PCA 降維空間中，計算每個細胞最近的 K 個鄰居（K-Nearest Neighbors），建立網絡連接。

計算 SNN 權重

用共享近鄰 (SNN)Shared Nearest Neighbors：兩個細胞共享的鄰居越多，它們的連接權重越高優化邊的權重——共享鄰居越多，連接越強。

社區發現 (Community Detection)

使用 Louvain 或 Leiden 演算法將網絡切割成群集。Leiden 是 Louvain 的改良版，產生更具連通性的社區，目前推薦使用 Leiden。

互動模擬

Resolution 參數的影響

Resolution控制聚類的細緻程度：低值產生少量大群，高值產生多個小群 是決定分群粗細的核心參數。拖動滑桿觀察同一組數據在不同 resolution 下的分群結果。

方法比較

Graph-based vs K-means

Graph-based（推薦）K-means
預設於Seurat、Scanpy傳統 ML
需預設 K？不需要（由 resolution 控制粗細）需要（必須事先指定群數）
群體形狀可處理非凸、不規則形狀假設球形、大小相近
適合 scRNA-seq？✅ 是⚠️ 通常不適合
程式碼

實作範例

# 建立 KNN/SNN 圖（使用前 15 個 PCs）
pbmc <- FindNeighbors(pbmc, dims = 1:15)

# Leiden 聚類（推薦）
pbmc <- FindClusters(pbmc, resolution = 0.5, algorithm = 4)  # algorithm=4 為 Leiden

# 或 Louvain（預設）
pbmc <- FindClusters(pbmc, resolution = 0.5)

# 嘗試多個 resolution 比較
pbmc <- FindClusters(pbmc, resolution = c(0.2, 0.5, 0.8, 1.2))
# 建立近鄰圖
sc.pp.neighbors(adata, n_pcs=15)

# Leiden 聚類（推薦）
sc.tl.leiden(adata, resolution=0.5)

# 或 Louvain
sc.tl.louvain(adata, resolution=0.5)
常見陷阱

聚類分析的三大陷阱

✂️ 過度聚類

Resolution 設太高，同一種細胞被拆成多群。檢查方法：群間 DE 基因極少或差異微小。

🧪 批次效應

聚類反映樣本來源而非細胞類型。解法：聚類前先做 Integration。

👯 Doublets

UMAP 上出現在兩大群之間的「橋樑」小群，marker 基因混合了兩種細胞特徵。需用 DoubletFinder 預先過濾。
