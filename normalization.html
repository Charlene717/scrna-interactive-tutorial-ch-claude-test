標準化 (Normalization)

消除技術偏差，讓不同細胞間的基因表達值具備可比性。

為什麼

為什麼需要標準化？

即使是完全相同的細胞，也會因為定序深度每個細胞在定序儀上獲得的總 reads 數不同、捕獲效率、液滴大小等技術因素，呈現不同的原始計數。

📊

定序深度

不同細胞獲得的總 reads 數不同。

🧲

捕獲效率

逆轉錄成 cDNA 的效率有隨機差異。

💧

液滴大小

微流控中每個液滴試劑量有微小差異。

🧬

RNA 含量差異

不同類型細胞攜帶的 RNA 絕對量不同。

互動模擬

數據分佈的變化

點擊按鈕觀察不同標準化方法如何改變分佈。

原始數據：嚴重右偏，大量零值。
方法比較

三種主要方法

特性LogNormalizeScranSCTransform
核心邏輯線性縮放 + log(x+1)細胞池解卷積正則化負二項回歸
速度極快中等較慢
適用情境快速探索細胞 RNA 含量差異大正式分析（目前主流）

🌳 我該用哪種方法？

Q1:
Seurat 正式分析？ → SCTransform。
Q2:
Scanpy？ → normalize_total + log1p。
Q3:
細胞大小差異極大？ → Scran。
程式碼

實作範例

# LogNormalize（預設）
pbmc <- NormalizeData(pbmc, normalization.method = "LogNormalize", scale.factor = 10000)
# SCTransform（推薦）
pbmc <- SCTransform(pbmc, vars.to.regress = "percent.mt", verbose = FALSE)
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
adata.raw = adata.copy()
💡
Normalization ≠ Scaling。Normalization 處理細胞間定序深度；Scaling 是基因做 Z-score。兩者不同。
