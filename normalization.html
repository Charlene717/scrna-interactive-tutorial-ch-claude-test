標準化 (Normalization)

消除技術偏差，讓不同細胞間的基因表達值具備可比性。

為什麼

為什麼需要標準化？

即使是完全相同的細胞，也會因為定序深度Sequencing Depth：每個細胞在定序儀上獲得的總 reads 數不同、捕獲效率Capture Efficiency：逆轉錄與文庫備製過程中的隨機差異、液滴大小等技術因素，呈現不同的原始計數。標準化的任務就是移除這些非生物學的變異，讓後續分析聚焦於真實的表達差異。

📊

定序深度

不同細胞獲得的總 reads 數不同，整體信號強度基線各異。

🧲

捕獲效率

逆轉錄成 cDNA 的效率有隨機差異，並非每個 RNA 分子都能被捕獲。

💧

液滴大小

微流控技術中，每個液滴包含的試劑量有微小差異。

🧬

RNA 含量差異

不同類型/大小的細胞本身攜帶的 RNA 絕對量就不同。

互動模擬

數據分佈的變化

點擊下方按鈕，觀察不同標準化方法如何改變基因表達值的分佈形態。

原始數據 (Raw Counts)：嚴重右偏，少數高表達基因主導分佈，大量零值。
方法比較

三種主要方法

特性LogNormalizeScranSCTransform
核心邏輯線性縮放 + log(x+1)細胞池解卷積正則化負二項回歸
速度極快中等較慢
處理零值能力一般較好最佳
適用情境快速探索細胞 RNA 含量差異大正式分析（目前主流）

🌳 我該用哪種方法？

Q1:
你用 Seurat 嗎？ → 是 → 預設 LogNormalize 可用於初步探索；正式分析建議 SCTransform。
Q2:
你用 Scanpy 嗎？ → 是 → 預設流程為 sc.pp.normalize_total + sc.pp.log1p（等同 LogNormalize）。
Q3:
數據中有極端大小不同的細胞？ → 是 → 考慮 Scran 的 pooling deconvolution。
程式碼

實作範例

# 方法一：LogNormalize（預設）
pbmc <- NormalizeData(pbmc, normalization.method = "LogNormalize", scale.factor = 10000)

# 方法二：SCTransform（推薦用於正式分析）
# 同時處理 normalization + feature selection + scaling
pbmc <- SCTransform(pbmc, vars.to.regress = "percent.mt", verbose = FALSE)
# 方法一：LogNormalize 等效做法
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)

# 儲存原始計數以備後用
adata.raw = adata.copy()
💡
Normalization ≠ Scaling。Normalization 處理的是細胞間定序深度差異；Scaling（下下步）是將基因做 Z-score 轉換，讓 PCA 不被高表達基因主導。兩者是不同步驟。
