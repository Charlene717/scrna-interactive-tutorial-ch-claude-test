Step 02

Normalization

不同細胞因捕獲效率或測序深度不同，導致 UMI 總數差異極大。 Normalization 讓這些細胞的基因表達值具有可比較性。

LogNormalize SCTransform NormalizeData
為什麼需要 Normalization？
📌 核心問題

細胞 A 測到 5,000 個 UMI，細胞 B 測到 20,000 個 UMI。若基因 X 在 A 中有 100 個 UMI、在 B 中有 200 個，兩者基因 X 的「真實表達量」是否相同？答案是：A 的 100/5000 = 2%，B 的 200/20000 = 1%，其實 A 表達更高。

R (Seurat) 程式碼
◆ R · Seurat
# 方法一：LogNormalize（預設，最廣泛使用）
# 公式：log1p(count / total_UMI * scale.factor)
seurat_obj <- NormalizeData(
  seurat_obj,
  normalization.method = "LogNormalize",
  scale.factor = 10000   # 統一縮放到每個細胞 10,000 counts
)

# 結果存在 seurat_obj[["RNA"]]@data（log 化的 counts）
# 原始 counts 保留在 seurat_obj[["RNA"]]@counts

# 確認：看看前幾個細胞的前幾個基因
seurat_obj[["RNA"]]@data[1:5, 1:5]
◆ R · SCTransform (v2)
# 方法二：SCTransform（使用 Pearson 殘差，更先進）
# 同時完成 Normalization + Scaling，且自動處理測序深度影響
# 安裝：install.packages("glmGamPoi")
library(glmGamPoi)

seurat_obj <- SCTransform(
  seurat_obj,
  vst.flavor = "v2",          # 推薦使用 v2
  vars.to.regress = "percent.mt",  # 可額外回歸掉粒線體效應
  verbose = FALSE
)

# 注意：使用 SCTransform 後，後續 PCA 等步驟會自動使用
# SCT assay，不需要再執行 FindVariableFeatures 和 ScaleData

# 查看 SCT assay 的資訊
seurat_obj[["SCT"]]
方法比較
方法原理優點注意事項
LogNormalize 除以 total UMI × 10,000，再取 log1p 快速、廣泛支援、易於理解 仍受測序深度影響
SCTransform v2 廣義線性模型（GLM）計算 Pearson 殘差 更準確校正技術效應、適合整合分析 計算較慢，需要額外套件
💡 何時選 SCTransform？

若你的分析涉及多樣本整合（Harmony/Seurat CCA），或資料中測序深度差異很大，建議使用 SCTransform v2。一般單樣本分析，LogNormalize 已足夠。
