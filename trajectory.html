軌跡分析 (Trajectory / Pseudotime)

從靜態的快照數據中推估細胞狀態轉變的動態路徑與時間順序。

核心概念

什麼是 Pseudotime？

scRNA-seq 是一張「快照」——在某個時間點捕捉所有細胞的狀態。但因為不同細胞處於分化過程的不同階段，我們可以利用它們之間的表達漸變，重建出一條擬時序 (Pseudotime)一個連續的數值軸，代表細胞沿某個生物學過程（如分化、活化）的進展程度軸——好比從一張團體照中推測每個人的年齡。

🎯

適用情境

細胞分化（幹細胞→終端細胞）、免疫細胞活化、疾病進程、藥物反應動態。

⚠️

前提假設

數據中必須存在處於不同狀態的細胞。如果所有細胞都已是終端分化狀態，就無法重建軌跡。

方法比較

主流工具

工具核心邏輯適用情境
Monocle3 (R)在 UMAP 上學習主圖 (Principal Graph)通用型，支援分支軌跡
Slingshot (R)在降維空間擬合同時主曲線簡單線性/分支軌跡，統計嚴謹
RNA Velocity (Py)利用 spliced/unspliced mRNA 比率推斷方向推斷轉變的「方向」，而非只是排序
CytoTRACE基於基因多樣性推估分化潛能預測哪些細胞較「年輕」（較未分化）
PAGA (Py)基於圖的抽象軌跡推斷大規模數據的粗粒度拓撲結構

🌳 我該用哪個方法？

Q1:
你只想知道分化的「順序」？ → Monocle3 或 Slingshot。
Q2:
你還想知道分化的「方向」？ → RNA Velocity（scVelo），能推斷每個細胞的瞬時方向。
Q3:
你有很多 cluster，想先看整體拓撲關係？ → PAGA，然後再做細粒度軌跡。
Q4:
你想知道哪些細胞最未分化？ → CytoTRACE 做分化潛能預測。
程式碼

實作範例

library(monocle3)

# 從 Seurat 轉換為 Monocle3 cell_data_set
cds <- as.cell_data_set(pbmc)

# 學習軌跡圖
cds <- learn_graph(cds)

# 指定起始細胞（root），計算 pseudotime
cds <- order_cells(cds)  # 互動選擇 root

# 視覺化
plot_cells(cds, color_cells_by = "pseudotime")

# 找沿軌跡變化的基因
trajectory_genes <- graph_test(cds, neighbor_graph = "principal_graph")
import scvelo as scv

# 讀入包含 spliced/unspliced 的 loom 或 h5ad
adata = scv.read("data.h5ad")

# 預處理
scv.pp.filter_and_normalize(adata, min_shared_counts=20)
scv.pp.moments(adata, n_pcs=30, n_neighbors=30)

# 估算 RNA velocity
scv.tl.velocity(adata)
scv.tl.velocity_graph(adata)

# 視覺化：在 UMAP 上畫箭頭
scv.pl.velocity_embedding_stream(adata, basis="umap")

# 找 velocity 驅動的基因
scv.tl.rank_velocity_genes(adata)
top_genes = adata.uns["rank_velocity_genes"]["names"]
注意事項

軌跡分析的注意事項

🚨
Pseudotime ≠ 真實時間。它是根據表達相似性排列的「虛擬」時間軸。不同的起始點設定、不同的方法，可能給出不同的軌跡。結果需要用實驗（如 lineage tracing、time-course 實驗）驗證。
⚠️
RNA Velocity 的限制：scVelo 依賴 spliced/unspliced 讀段的準確量化，這在 10x Chromium 3' 數據中可能不夠精確。近年來 UniTVelo 和 VeloVAE 等新方法嘗試改善此問題。
