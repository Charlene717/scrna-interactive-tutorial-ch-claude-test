STEP 5 / 9

主成分分析 (PCA)

將數千個基因維度壓縮為數十個主成分，保留關鍵生物學訊號、過濾隨機噪音。

核心概念

PCA 的角色

即使只使用 2000 個 HVGs，數據仍然是 2000 維的——這對聚類和視覺化來說太高了（維度災難Curse of Dimensionality：高維空間中，距離計算變得不可靠，點與點之間的距離趨於相同）。PCA 找出數據中方差最大的方向（主成分），用前 N 個主成分就能捕捉絕大部分的真實訊號，而後面的主成分多半是噪音。

🗜️

降維

從 ~2000 維壓縮到 10–50 維。

🧹

去噪

丟棄低方差的主成分，等同過濾噪音。

⚡

加速

降維後的聚類、UMAP 計算速度大幅提升。

互動模擬

Elbow Plot：選擇多少個主成分？

Elbow Plot 顯示每個主成分解釋的方差比例。你需要找到「手肘」——曲線開始趨平的位置，在此之後的主成分貢獻微小。拖動滑桿設定你的選擇。

🌳 如何決定 nPCs？

方法1:
Elbow Plot — 選在曲線趨平處，通常 10–20。
方法2:
JackStraw test — 統計檢定每個 PC 的顯著性（計算較慢）。
經驗值:
大多數情況 15–30 即足夠。稍微選多一些比選少安全（寧可包含一點噪音，也不要遺漏訊號）。
程式碼

實作範例

# 執行 PCA
pbmc <- RunPCA(pbmc, features = VariableFeatures(pbmc))

# 視覺化前兩個主成分
DimPlot(pbmc, reduction = "pca")

# Elbow Plot 決定 nPCs
ElbowPlot(pbmc, ndims = 50)

# 可選：JackStraw 統計檢定
pbmc <- JackStraw(pbmc, num.replicate = 100)
pbmc <- ScoreJackStraw(pbmc, dims = 1:30)
JackStrawPlot(pbmc, dims = 1:30)
# 執行 PCA
sc.tl.pca(adata, n_comps=50)

# 視覺化
sc.pl.pca(adata)

# Elbow Plot（方差比例圖）
sc.pl.pca_variance_ratio(adata, n_pcs=50)
