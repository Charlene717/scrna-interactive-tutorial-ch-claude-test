Step 01

Quality Control

過濾低品質細胞、空液滴與雙細胞（doublets）。QC 是整個分析流程的基石—— 跳過它，後續所有結果都將失去可信度。

nFeature_RNA nCount_RNA percent.mt DoubletFinder SoupX
核心 QC 指標
🧬

nFeature_RNA

單一細胞偵測到的基因數量。
過低 <200：破裂細胞或空液滴
過高：可能為雙細胞 (doublet)

📊

nCount_RNA

所有轉錄本分子的總 UMI 數量。應與 nFeature 呈高度正相關。異常值通常對應空液滴或多細胞。

🔴

percent.mt

粒線體基因佔比。細胞死亡時細胞質 mRNA 外漏，粒線體 RNA 相對殘留。常見閾值：5%–20%（視組織而異）。

互動式 QC 模擬器

拖曳滑桿調整閾值，即時觀察過濾效果。綠色 = 通過；紅色 = 剔除。

R (Seurat) 程式碼
◆ R · Seurat 
# 載入套件
library(Seurat)
library(ggplot2)

# 讀取 10x Genomics 資料
counts <- Read10X(data.dir = "filtered_feature_bc_matrix/")
seurat_obj <- CreateSeuratObject(
  counts = counts,
  project = "MyProject",
  min.cells = 3,     # 基因至少出現在 3 個細胞
  min.features = 200  # 細胞至少有 200 個基因
)

# 計算粒線體基因比例
seurat_obj[["percent.mt"]] <- PercentageFeatureSet(
  seurat_obj, pattern = "^MT-"  # 人類用 ^MT-，小鼠用 ^mt-
)

# 視覺化 QC 指標
VlnPlot(seurat_obj,
  features = c("nFeature_RNA", "nCount_RNA", "percent.mt"),
  ncol = 3, pt.size = 0
)

# 過濾低品質細胞
seurat_obj <- subset(seurat_obj,
  subset = nFeature_RNA > 200 &
           nFeature_RNA < 4000 &
           percent.mt   < 10
)

# 確認過濾後的細胞數量
cat("保留細胞數：", ncol(seurat_obj), "\n")
◆ R · DoubletFinder 
# 安裝與載入 DoubletFinder
# remotes::install_github("chris-mcginnis-ucsf/DoubletFinder")
library(DoubletFinder)

# 先進行標準前處理（詳見後續步驟）
seurat_obj <- seurat_obj |>
  NormalizeData() |>
  FindVariableFeatures() |>
  ScaleData() |>
  RunPCA() |>
  RunUMAP(dims = 1:20)

# 參數掃描：找最佳 pK 值
sweep_res <- paramSweep(seurat_obj, PCs = 1:20, sct = FALSE)
sweep_stats <- summarizeSweep(sweep_res, GT = FALSE)
pk_val <- find.pK(sweep_stats)
optimal_pK <- as.numeric(as.character(pk_val$pK[which.max(pk_val$BCmetric)]))

# 預估 doublet 比例（依據 10x 官方表格）
nExp <- round(0.075 * nrow(seurat_obj@meta.data))

# 執行 DoubletFinder
seurat_obj <- doubletFinder(seurat_obj,
  PCs = 1:20, pN = 0.25,
  pK = optimal_pK, nExp = nExp, reuse.pANN = FALSE, sct = FALSE
)

# 移除 doublets
df_col <- grep("DF.classification", colnames(seurat_obj@meta.data), value = TRUE)
seurat_obj <- subset(seurat_obj, cells = rownames(seurat_obj@meta.data[seurat_obj@meta.data[[df_col]] == "Singlet",]))
cat("移除 doublets 後：", ncol(seurat_obj), "個細胞\n")
◆ R · SoupX 
# 安裝：install.packages("SoupX")
library(SoupX)

# 讀取 10x 原始輸出（需 raw + filtered 兩個資料夾）
sc <- load10X("path/to/10x_output/")

# 自動估計環境 RNA 污染率
sc <- autoEstCont(sc)

# 查看估計的污染率
cat("估計污染率：", sc$fit$rhoEst * 100, "%\n")

# 校正矩陣
out <- adjustCounts(sc, roundToInt = TRUE)

# 用校正後的矩陣建立 Seurat 物件
seurat_obj <- CreateSeuratObject(
  counts = out,
  project = "SoupX_corrected"
)
QC 策略比較
策略固定閾值（傳統）自適應閾值（最佳實踐）
概念主觀的固定數值（如 MT% < 5%）根據資料分佈決定（Data-driven）
方法文獻經驗值絕對中位差（MAD）：中位數 ± 3×MAD
R 函數subset(obj, percent.mt < 5)scater::isOutlier()
缺點多樣本或特殊組織容易出錯對不同樣本品質差異具強健性
💡 核心建議

沒有放諸四海皆準的閾值。心肌細胞的粒線體比例天然偏高，血液樣本要額外過濾紅血球基因（HBA, HBB）。永遠先看小提琴圖與散佈圖，再決定閾值。

⚠️ 避免 Over-filtering

過嚴的過濾可能把靜止期細胞或罕見亞群一起濾掉。建議先寬鬆過濾、分群後，若發現某 cluster 指標異常再回頭移除。QC 是迭代過程。
