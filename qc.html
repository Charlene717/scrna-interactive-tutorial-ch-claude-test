STEP 1 / 9

品質管控 (Quality Control)

過濾低品質細胞與技術假象，為後續分析奠定乾淨的數據基礎。

概覽

為什麼 QC 是最關鍵的第一步？

單細胞捕獲技術的物理限制意味著原始數據中必然混入大量雜訊——空液滴Empty droplets：未包含細胞的液滴，只捕獲了環境背景 RNA、破裂的死細胞、以及雙細胞 (Doublets)兩個細胞被包裹在同一個液滴中，產生混合的基因表達譜。若未妥善過濾，這些雜訊會在降維與分群時產生虛假的 cluster，導致錯誤的生物學結論。

💡
核心原則：QC 沒有「放諸四海皆準」的閾值。最佳做法是根據你的物種、組織類型和定序平台，結合視覺化圖表來做判斷，而非盲目套用他人的數字。
核心指標

細胞層級的三大 QC 指標

🔢

nFeature_RNA

單一細胞中偵測到的基因數量。過低（< 200）代表空液滴或碎片；過高則暗示 doublet。

📊

nCount_RNA

該細胞的總 UMI 分子數。應與 nFeature 高度正相關，異常偏離者通常為技術假象。

🔋

percent.mt

粒線體基因佔比，是判斷細胞死活最重要的指標。細胞破裂時 mRNA 外洩，但粒線體 RNA 傾向殘留。

基因層級

基因層級的過濾

除了細胞品質，我們也需移除極低表達的基因——通常剔除「在少於 3 個細胞中被偵測到的基因」。這些基因多半是定序雜訊，且在統計分析中無法提供顯著性。

🧬 核糖體基因（RPS / RPL）

比例過低可能暗示細胞品質不佳。部分流程會選擇性移除或標記，但並非普遍做法。

🩸 紅血球基因（HBA / HBB）

在血液或富含血管的組織中，若非研究紅血球本身，通常會過濾掉比例過高的細胞。

進階處理

處理技術性假象

👯 雙細胞偵測 (Doublet Detection)

基本的 nFeature 上限無法完全抓到由兩個小細胞組成的 doublet。需要使用專門的演算法來模擬 doublet 特徵並計算每個細胞的 doublet score。

常用工具：DoubletFinder (R)、Scrublet (Python)、scDblFinder (R/Bioconductor)

🫧 環境背景 RNA (Ambient RNA / Soup)

破裂細胞釋出的游離 RNA 會被其他液滴一併捕獲，導致「污染」——例如 T 細胞竟然檢測到肝細胞特異基因。需從表達矩陣中估算並扣除這部分背景。

常用工具：SoupX (R)、CellBender (Python / GPU)

決策引導

如何選擇 QC 閾值？

🌳 閾值策略決策流程

Q1:
你有多個樣本 / 批次嗎？ → 是 → 建議使用自適應閾值（MAD-based），讓每個樣本根據自身分佈決定閾值。
Q2:
你的組織有特殊 MT% 特徵嗎？（如心肌細胞 MT% 天然偏高）→ 是 → 不要套用通用的 5% 閾值，需根據組織背景調高。
Q3:
是否關注稀有細胞群？ → 是 → 建議寧可寬鬆；過嚴的過濾可能將靜止期 / 小型細胞誤判為低品質而移除。
Q4:
初次分析？ → 是 → 先用寬鬆閾值做初步分群，如果發現某 cluster 全是高 MT% 且無特異 marker，再回頭收緊。
策略固定閾值（傳統）自適應閾值（推薦）
概念使用經驗數值，如 MT% < 5%基於數據分佈自動決定
常見做法文獻常規值中位數 ± 3 × MAD
代表工具Seurat 手動設定scater isOutlier()
適用情境單樣本、快速探索多樣本整合、正式分析
程式碼

QC 實作範例

# 讀入 10X 數據
library(Seurat)
pbmc.data <- Read10X(data.dir = "filtered_feature_bc_matrix/")
pbmc <- CreateSeuratObject(counts = pbmc.data, min.cells = 3, min.features = 200)

# 計算 MT%
pbmc[["percent.mt"]] <- PercentageFeatureSet(pbmc, pattern = "^MT-")

# 視覺化 QC 指標（先觀察再決定閾值！）
VlnPlot(pbmc, features = c("nFeature_RNA", "nCount_RNA", "percent.mt"), ncol = 3)

# 過濾（依觀察結果調整數值）
pbmc <- subset(pbmc,
  subset = nFeature_RNA > 200 & nFeature_RNA < 4000 & percent.mt < 10
)
import scanpy as sc

# 讀入數據
adata = sc.read_10x_mtx("filtered_feature_bc_matrix/", var_names="gene_symbols")

# 基因層級過濾：至少在 3 個細胞中表達
sc.pp.filter_genes(adata, min_cells=3)

# 計算 QC 指標
adata.var["mt"] = adata.var_names.str.startswith("MT-")
sc.pp.calculate_qc_metrics(adata, qc_vars=["mt"], inplace=True)

# 視覺化（先觀察！）
sc.pl.violin(adata, ["n_genes_by_counts", "total_counts", "pct_counts_mt"])

# 過濾
adata = adata[adata.obs["n_genes_by_counts"] > 200]
adata = adata[adata.obs["n_genes_by_counts"] < 4000]
adata = adata[adata.obs["pct_counts_mt"] < 10]
⚠️
先看圖再設閾值！上面的數字（200 / 4000 / 10%）只是示範。實際分析時務必先執行 VlnPlot 或 sc.pl.violin 觀察數據分佈後再決定。

📝 自我檢測

1. 某細胞的 nFeature_RNA 極高、nCount_RNA 也極高，最可能的原因是？

A. 該細胞正在大量轉錄基因
B. 定序深度特別深的正常細胞
C. 可能是 doublet（雙細胞）
D. 粒線體基因佔比過高

2. 你正在分析心肌組織的 scRNA-seq 數據，發現大量細胞的 MT% 介於 8–15%。最佳做法是？

A. 仍然使用 5% 作為閾值，嚴格過濾
B. 根據組織特性調高閾值，因為心肌細胞粒線體含量天然偏高
C. 移除所有 MT 相關基因，問題就解決了
D. 不需要做 QC，直接進入下游分析
