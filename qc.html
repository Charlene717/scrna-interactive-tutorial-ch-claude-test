STEP 1 / 9

品質管控 (Quality Control)

過濾低品質細胞與技術假象，為後續分析奠定乾淨的數據基礎。

概覽

為什麼 QC 是最關鍵的第一步？

單細胞捕獲技術的物理限制意味著原始數據中必然混入大量雜訊——空液滴Empty droplets：未包含細胞的液滴，只捕獲了環境背景 RNA、破裂的死細胞、以及雙細胞 (Doublets)兩個細胞被包裹在同一個液滴中，產生混合的基因表達譜。若未妥善過濾，這些雜訊會在降維與分群時產生虛假的 cluster，導致錯誤的生物學結論。

💡
核心原則：QC 沒有放諸四海皆準的閾值。根據你的物種、組織類型和定序平台，結合視覺化圖表來做判斷。
核心指標

細胞層級的三大 QC 指標

🔢

nFeature_RNA

單一細胞中偵測到的基因數量。過低代表空液滴；過高暗示 doublet。

📊

nCount_RNA

該細胞的總 UMI 分子數。應與 nFeature 高度正相關。

🔋

percent.mt

粒線體基因佔比，判斷細胞死活最重要的指標。

進階處理

處理技術性假象

👯 雙細胞偵測

基本閾值無法完全抓到 doublet。常用工具：DoubletFinder (R)、Scrublet (Python)、scDblFinder (R)

🫧 環境背景 RNA (Soup)

破裂細胞的游離 RNA 污染。常用工具：SoupX (R)、CellBender (Python)

決策引導

如何選擇 QC 閾值？

🌳 閾值策略決策流程

Q1:
多個樣本？ → 是 → 使用自適應閾值（MAD-based）。
Q2:
組織有特殊 MT%？（如心肌細胞）→ 是 → 根據背景調高閾值。
Q3:
關注稀有細胞？ → 是 → 寧可寬鬆。
程式碼

QC 實作範例

library(Seurat)
pbmc.data <- Read10X(data.dir = "filtered_feature_bc_matrix/")
pbmc <- CreateSeuratObject(counts = pbmc.data, min.cells = 3, min.features = 200)
pbmc[["percent.mt"]] <- PercentageFeatureSet(pbmc, pattern = "^MT-")
VlnPlot(pbmc, features = c("nFeature_RNA", "nCount_RNA", "percent.mt"), ncol = 3)
pbmc <- subset(pbmc, subset = nFeature_RNA > 200 & nFeature_RNA < 4000 & percent.mt < 10)
import scanpy as sc
adata = sc.read_10x_mtx("filtered_feature_bc_matrix/", var_names="gene_symbols")
sc.pp.filter_genes(adata, min_cells=3)
adata.var["mt"] = adata.var_names.str.startswith("MT-")
sc.pp.calculate_qc_metrics(adata, qc_vars=["mt"], inplace=True)
sc.pl.violin(adata, ["n_genes_by_counts", "total_counts", "pct_counts_mt"])
adata = adata[(adata.obs.n_genes_by_counts > 200) & (adata.obs.n_genes_by_counts < 4000) & (adata.obs.pct_counts_mt < 10)]
